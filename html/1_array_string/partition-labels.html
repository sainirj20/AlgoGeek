763. Partition Labels Link to LeetCode You are given a string s. We want to partition the string into as many parts as possible so that each letter appears in at most one part. For example, the string "ababcc" can be partitioned into ["abab", "cc"], but partitions such as ["aba", "bcc"] or ["ab", "ab", "cc"] are invalid. Note that the partition is done so that after concatenating all the parts in order, the resultant string should be s. Return a list of integers representing the size of these parts. Input: s = "ababcbacadefegdehijhklij" Output: [9,7,8] Explanation: The partition is "ababcbaca", "defegde", "hijhklij". This is a partition so that each letter appears in at most one part. A partition like "ababcbacadefegde", "hijhklij" is incorrect, because it splits s into less parts. public List< Integer> partitionLabels(String S) { LinkedHashMap< Character, int[]> map = new LinkedHashMap<>(); for (int i = 0; i < S.length(); i++) { char c = S.charAt(i); int[] range = map.getOrDefault(c, new int[] {i, i}); range[1] = i; map.put(c, range); } List result = new ArrayList<>(); int[] prev = null; for (Map.Entry entry : map.entrySet()) { int[] curr = entry.getValue(); if (prev == null) { prev = curr; } else { if (curr[0] < prev[1]) { prev[1] = Math.max(curr[1], prev[1]); } else { result.add(prev[1] - prev[0] + 1); prev = curr; } } } result.add(prev[1] - prev[0] + 1); return result; }
